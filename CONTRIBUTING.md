# Contributing

Thank you for your interest in contributing.

This project aims to make low-level programming more accessible through interactive coding challenges, MCQs, and a browser-based playground.

## Before You Start

* Check existing issues before opening a new one.
* Keep pull requests focused on a single change.
* Write clear commit messages.
* Ensure your code is tested and follows the existing style.

## Reporting Issues

When reporting a bug, include:

* A clear description of the problem.
* Steps to reproduce it.
* Expected and actual behavior.
* Relevant logs or screenshots, if applicable.

## Pull Requests

* Fork the repository and create a new branch.
* Make your changes.
* Verify everything works as expected.
* Submit a pull request describing what you changed and why.

Every contribution—whether it's improving the platform, fixing bugs, enhancing the playground, or refining the user experience—is appreciated.
